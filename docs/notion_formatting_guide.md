# Notion 富文本格式规范

本文档为 AI 日报撰写编辑（writer agent）提供 Notion 专属 Markdown 方言的格式说明。
输出的 Markdown 将通过程序自动转换为 Notion blocks 并上传，因此需要遵循本规范。

---

## 一、支持的块类型（Block Types）

### 1. 标准 Markdown（通用）

| 语法 | Notion Block |
|------|-------------|
| `# 标题` | heading_1 |
| `## 标题` | heading_2 |
| `### 标题` | heading_3 |
| `#### 标题` | heading_3（等同 ###）|
| `> 引用` | quote |
| `- 条目` | bulleted_list_item |
| `1. 条目` | numbered_list_item |
| `---` | divider |
| `![](url)` | image（仅支持 http/https） |
| `**文字**` | 粗体（bold annotation）|
| `*文字*` | 斜体（italic annotation）|
| `[文字](url)` | 超链接 |

### 2. 代码块

````
```python
# 代码内容
x = 1 + 2
```
````

→ 转换为 Notion code block，支持语言：`python`, `javascript`, `typescript`, `bash`, `json`, `yaml`, `markdown`，其他均以 `plain text` 处理。

### 3. 数学公式块（Equation）

```
$$
\mathcal{L} = \mathbb{E}_{x,t}\left[\|\epsilon - \epsilon_\theta(x_t, t)\|^2\right]
$$
```

→ 转换为 Notion equation block（LaTeX 格式）。

### 4. Callout 块

```
> [!TYPE] 文字内容
```

`TYPE` 可选值及对应样式：

| TYPE | 图标 | 背景色 | 适用场景 |
|------|------|--------|----------|
| `KEY` | 🔑 | 紫色 | 核心贡献、关键创新 |
| `TIP` | 💡 | 绿色 | 亮点提示、实用技巧 |
| `GOAL` | 🎯 | 橙色 | 研究目标、动机 |
| `RESULT` | 📊 | 黄色 | 关键结果、数据指标 |
| `METHOD` | 🔧 | 灰色 | 方法说明 |
| `NOTE` | 📝 | 蓝色 | 备注信息 |
| `INFO` | ℹ️ | 蓝色 | 背景信息 |
| `WARNING` | ⚠️ | 红色 | 警告、局限性 |
| `PAPER` | 📄 | 默认 | 论文相关 |

**示例：**
```
> [!KEY] 本文首次提出将扩散模型与流匹配统一在同一理论框架下，显著提升采样效率。
> [!GOAL] 解决现有文本到图像模型在细粒度语义控制方面的不足。
> [!RESULT] 在 GenEval 基准上达到 0.87，超越 DALL·E 3 等主流模型。
```

### 5. Toggle 折叠块

```
<toggle>标题文字
内容行 1
内容行 2
- 列表项
</toggle>
```

→ 转换为 Notion toggle block，内容默认折叠。标题显示，点击后展开内容。

**注意：**
- `<toggle>` 与标题文字在同一行（无空格）
- 内容可包含任意支持的 Markdown 语法
- 以 `</toggle>` 单独一行结束

**示例：**
```
<toggle>🔧 方法详解
**核心架构**：Transformer-based diffusion model

- 创新点 1：引入 Flow Matching 目标函数替代传统 DDPM
- 创新点 2：自适应时间步调度策略
- 创新点 3：无分类器引导的条件控制
</toggle>
```

### 6. 行内格式

| 语法 | 效果 |
|------|------|
| `` `代码` `` | 行内代码（code annotation）|
| `==高亮文字==` | 黄色高亮背景 |
| `**粗体**` | 粗体 |
| `*斜体*` | 斜体 |
| `[文字](url)` | 超链接 |

## 二、论文条目格式规范

对于 AIGC 学术前沿和 HCI 前沿的每篇论文，使用以下丰富格式呈现：

```markdown
#### [论文标题](arXiv链接)

> [!KEY] 核心贡献：一句话概括本文最大创新点（突出与现有工作的差异）

**作者**：xxx et al. | **发布**：YYYY-MM-DD | **领域**：图像生成 / LLM / HCI 等

**研究动机**：当前方法存在什么问题，本文为何要解决这个问题（1-2句）

<toggle>🔧 方法详解
**核心框架/模型名称**：XXX

- **组件 1**：描述其功能与创新
- **组件 2**：描述其功能与创新
- **训练策略**：说明关键训练细节（如有）
</toggle>

<toggle>📊 实验与结果
- **评估基准**：XXX benchmark / XXX dataset
- **关键指标**：达到 XXX，相比 [baseline] 提升 XX%
- **重要发现**：消融实验 / 用户研究的核心结论
</toggle>

**论文预览**：[HuggingFace Papers](HF URL)
```

## 三、新闻条目格式规范

```markdown
### [新闻标题](来源链接)
![](图片URL)
> [!INFO] 核心要点：一句话概括最重要的信息

新闻摘要（2-3句中文，说明事件详情、意义、行业影响）
```

## 四、工具条目格式规范

```markdown
### [工具名称](官网/GitHub链接)
> [!TIP] 核心功能：一句话描述工具的核心价值

**功能**：详细介绍工具能做什么
**适用**：目标用户群体与使用场景
**费用**：免费 / 付费（价格）/ 开源（协议）
```

## 五、HCI 条目格式规范

```markdown
### [工作标题](链接)
> [!GOAL] 研究问题：本工作聚焦解决的核心问题

**来源**：CHI 2025 / UIST 2025 / arXiv
**内容**：简洁中文介绍，包括方法特色与实际应用价值（2-3句）
```

## 七、重要约束

1. **链接真实性**：只使用原始数据中出现的真实链接，严禁虚构 URL
2. **图片条件**：仅当原始数据中含「图片URL」字段时才嵌入 `![](url)`，无则不加占位符
3. **论文预览**：仅当原始数据含「论文预览（含主图）」URL 时才添加预览链接
4. **Callout 简洁性**：每个 callout 的文字控制在 50 字以内，突出最关键信息
5. **Toggle 内容完整性**：Toggle 内的内容要有实质信息，不要空洞填充
6. **语言**：所有解读、摘要、分析必须使用中文；技术术语可保留英文
7. **不输出多余内容**：最终输出为纯 Markdown，不含解释性说明、代码围栏包裹等
